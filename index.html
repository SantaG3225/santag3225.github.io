Surge 5 iOS Rule Sets

SantaG Surge Rules

Personal Surge rule sets for iOS, optimized for a Malaysia-based setup with a U.S. proxy node.

Rule Sets

File Purpose Suggested Policy
ai.list AI services and related domains. Proxy
global.list Global services, developer tools, Google, YouTube, GitHub, and selected media platforms. Proxy
china-direct.list China mainland domains and services. DIRECT
edu.list Education websites, school portals, SSO, Microsoft login, Canvas, and MFA services. Proxy
private.list Private or custom rules. Depends on setup

Surge Configuration

Use these remote rule-set URLs in the [Rule] section of your Surge profile.

FINAL,DIRECT

Recommended Routing Logic

Education / AI / GitHub / Google / YouTube -> U.S. proxy
China mainland services                  -> DIRECT
Malaysia local services                  -> DIRECT
Unknown traffic                          -> DIRECT

This setup does not provide a China mainland return route. Services that require a China mainland IP still need a China mainland or return-home proxy node.

Rule Format

Each .list file contains Surge-compatible rules without policy names.

# Comment
DOMAIN-SUFFIX,example.com
DOMAIN,login.example.com
DOMAIN-KEYWORD,example
IP-CIDR,1.1.1.0/24

Validation

GitHub Actions validates the rule files automatically.

  • Unsupported rule types
  • Missing rule values
  • Duplicate rules inside the same file
  • Invalid domain values
  • CRLF line endings or missing final newline

Provided for personal routing and learning purposes. Follow local laws, platform terms, and network policies.
